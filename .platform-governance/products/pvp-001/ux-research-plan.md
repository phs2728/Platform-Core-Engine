# UX Research Plan — PVP-001 Phase 4

> **Phase**: 4 of 14. Find how customers move from Anxiety → Action on a hostel booking page.

---

## 1. UX research objectives

PVP-001 UX research produces the **information architecture**, **navigation patterns**, and **interaction flow** required for:

- Each Trust Stage to have a clear UI affordance.
- Each hypothesis to have a corresponding UX variant that can be A/B tested.
- Each persona to feel "this hostel is for me" within 5 seconds of landing.

---

## 2. Research questions (tied to hypotheses)

| Research question | Hypothesis |
|---|---|
| What information on the booking page most reduces booking anxiety? | HYP-PVP-001-001 |
| Which room-detail-page sections most influence the booking decision? | HYP-PVP-001-002 |
| How does CTA placement interact with user's attention? | HYP-PVP-001-003 |
| Which trust evidence items matter most, and where? | HYP-PVP-001-004 |

---

## 3. Personas revisited (UX-relevant attributes)

| Persona | UX-relevant traits | Primary Trust Stage |
|---|---|---|
| P1 Solo | Skims, scrolls deep; wants social proof and photos | Anxiety |
| P2 Couple | Reads carefully, values quiet and clean; sensitive to cancellation | Confidence |
| P3 Group | Plans collectively; needs comparison views | Discovery |
| P4 Long-stay | Wants detail (wifi, desk, weekly rate); reads FAQ | Confidence |
| P5 Family | Filters for cribs, family policy, neighborhood map | Evaluation |

## 4. Methods (Phase 4 only)

| Method | Output | Owner |
|---|---|---|
| Heuristic evaluation of competitor hostels (5 sites) | Competitive UX matrix | UX |
| Card sort (open, n=12) | Information architecture | UX + Research |
| Tree test (n=12) | Navigation findability | UX |
| First-click test (n=8) | Above-fold hierarchy | UX |
| Persona journey mapping (n=5) | Stage-by-stage emotional maps | UX + Research |
| Wireframing (low-fi) | Wireframe set | UX |
| Prototype (mid-fi) | Clickable prototype | UX + Frontend |
