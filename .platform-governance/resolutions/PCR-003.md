## 2. Operating interpretation (non-normative)

This section is **not** part of the Resolution text. It records operating guidance for execution.

### 2.1 Mode re-orientation

PCR-002 declared "Platform Development → Product Development". PCR-003 re-orients further: "Product Development → Client Project Delivery". The product (e.g., a hostel website) is a **client deliverable**, not an internal Platform extension. The Platform exists to serve the client.

### 2.2 What this changes vs. PCR-002

| PCR-002 (Product Lab Mode) | PCR-003 (Client Project Mode) |
|---|---|
| Phases 1–5 manual; Phases 6/7 manually coded | All 16 internal phases auto-executed by Platform Agency |
| Pre-Coding Plan gates Phase 6/7 coding | Pre-Coding Plan gates Client Project auto-execution |
| Customer is 사장님-as-founder | Customer is the external client (e.g., Envoy Hostel owner) — but for PVP-001, the client IS 사장님 as Platform Owner + Envoy Hostel project principal |
| Success metric = Hypothesis advance | Success metric = Premium $10k+ client deliverable PLUS Hypothesis advance |
| Output = Phase deliverables | Output = Client-facing delivery checklist (8 visible items) |

Full charter at `.platform-governance/clients/envoy-hostel-tours/CHARTER.md`.

### 2.5 Customer-facing delivery checklist (8 visible outputs)

The 8 client-visible outputs the customer (Envoy Hostel & Tours) will receive upon delivery:

1. **Beautiful design** — per 사장님 확립 design 헌장 (Kinfolk/Aman-grade, Pretendard, qvevri-wine).
2. **Fast speed** — Lighthouse mobile ≥ 85; TTFB P95 < 250ms.
3. **Booking-ready structure** — search → detail → booking → confirmation; payment via `payment` engine; A/B-tested CTA.
4. **Easy-to-manage CMS** — content via `cms` engine (page copy, room descriptions, FAQ); no-code editing by hostel staff.
5. **AI chatbot** — simple, brand-tone; reads content + booking state; answers FAQs (not in MVP scope, may be v1.1 iteration per PCR-003 §1 visible outputs).
6. **SEO** — Schema.org LodgingBusiness; canonical URLs; metadata; sitemap; Google Search Console verified.
7. **Admin pages** — for staff to manage rooms, prices, reviews, bookings; via Platform's existing admin UI patterns.
8. **Mobile experience** — fully responsive; PWA-ready; iOS Safari + Android Chrome verified.

The customer does NOT see: PRD, TRD, Agency OS, QES, Skill Library, Playbooks, Platform internals.
