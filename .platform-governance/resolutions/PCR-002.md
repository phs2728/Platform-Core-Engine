### 2.2 14-phase Product Development pipeline (canonical)

The verbatim PHASES block defines the canonical order. Order shall not be reversed.

| # | Phase | Output | Owner |
|---|---|---|---|
| 1 | Discovery Beta | Discovery Beta Checklist PASS | Product Manager |
| 2 | PRD | PRD (or update) | Product Manager + CEO |
| 3 | TRD | TRD | Tech Lead + Product Manager |
| 4 | UX | UX Research Plan + UX Output | UX |
| 5 | Visual Design | Visual Design Output | UI Design |
| 6 | Frontend | Frontend Implementation | Frontend |
| 7 | Backend | Backend Implementation | Backend |
| 8 | QA | QA Plan + Report | QA |
| 9 | QES | QES Compliance Report | QA / Platform Guardian |
| 10 | Deployment | Deployed URL + Release Note | Backend + DevOps |
| 11 | Analytics | Analytics implementation + dashboard | Analytics |
| 12 | Customer Interview | Interview records | Research |
| 13 | Evidence Collection | Evidence Records (validated) | Analytics + Research |
| 14 | Platform Learning | Promotion candidates + ADR proposals (if any) | Product Manager + PVC |

**Note**: Phases 6 and 7 (Frontend + Backend) require Pre-Coding Plan = `pvc-approved` per Constitution §17 / Product Lab Standard operative §3. Until that gate passes, Phases 1–5 (and revision of plans) are the only allowed activities.

### 2.5 Agent role reconfiguration (verbatim § "에이전트의 역할도 바뀝니다")

The verbatim block re-designates agent roles. The hierarchy is:

```
CEO
  │
Product Manager
  │
Research
  │
UX
  │
UI Design
  │
Frontend
  │
Backend
  │
SEO
  │
Marketing
  │
QA
  │
Analytics
```

These roles are **Product roles**, not Platform roles. They are populated in the **Agent Role Matrix** at `.platform-governance/products/pvp-001/agent-role-matrix.md` (companion document to this Resolution).

CEO owns vision, market, capital. PM owns scope and schedule. Research owns customer truth. UX owns flow and information. UI Design owns visual surface. Frontend / Backend own implementation. SEO owns discoverability. Marketing owns growth and conversion. QA owns acceptance. Analytics owns measurement and the Evidence Collection Plan.

### 2.6 Success criteria (verbatim § "SUCCESS")

> "Platform success is not shipping software.
> Platform success is shipping successful products."

Operationally, **Platform success** = at least one Product Lab has produced a Beta PASS with production evidence (per EPS ladder) promoting at least one validatable capability. Without Beta PASS, the Platform has not earned the title "Operating System" — it is only "Foundation".
