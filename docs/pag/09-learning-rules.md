# PAG Part 9 — Learning Rules

> Every completed project must update Learning Engine. Only evidence-based updates.

## Learning Feedback Loop

```
Project Completed
       ↓
  Collect Metrics
  (conversion, bounce, CTR, scroll depth, A/B results)
       ↓
  Knowledge Evolution Tracker
       ↓
  ┌─────┴──────┐
  ▼            ▼
Pattern     Anti-Pattern
Found        Found
  │            │
  ▼            ▼
  Knowledge Base
  (store as KnowledgeAsset)
       ↓
  Update:
  - Skills (confidence adjusted)
  - Skill Packs (new skills added)
  - Playbooks (refined sections)
  - QES (rules adjusted)
  - Executive Memory (new patterns)
       ↓
  Agency OS uses updated knowledge
  for next project
```

## What Gets Updated

| Artifact | Update Trigger | Evidence Required |
|---|---|---|
| Skill confidence | Production performance data | Metrics from ≥1 project |
| Skill Pack composition | New pattern discovered | A/B test winner |
| Playbook section | Section underperforming | Bounce rate > 70% |
| QES rule | Industry standard changed | Authoritative source update |
| Executive Memory | Success/failure pattern | Production evidence |

## Forbidden Learning Patterns

- ❌ Updating QES without evidence (minimum 10 chars evidence)
- ❌ Creating Skills from opinions
- ❌ Modifying Playbooks without production data
- ❌ Storing unverified community tips as Knowledge Assets

## Knowledge Asset Lifecycle

```
Created (Experimental)
    ↓ (evidence accumulates)
Upgraded (Community Proven → Enterprise Proven → Production Proven)
    ↓ (if disproven)
Deprecated → Rejected
```
