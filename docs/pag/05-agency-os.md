# PAG Part 5 — Agency OS

> How Agency OS orchestrates everything.

## Executive Layer (6 Agents)

| Agent | Role | When Active |
|---|---|---|
| **CEO Agent** | Goal definition, priority, scope | Phase 1 (Planning) |
| **Project Manager** | Task breakdown, work plan, dependencies | Phase 1-2 |
| **Agency Orchestrator** | Swarm creation, parallel execution, merge, retry | Phase 2-11 |
| **Memory Manager** | Executive Memory read/write | All phases |
| **Quality Director** | QES enforcement, review coordination | Phase 11-12 |
| **Release Director** | Release gate, deployment authorization | Phase 12 |

## Swarm Lifecycle

```
1. Agency Orchestrator creates Swarm (type determined by Phase)
2. Swarm Leader assigns tasks to Specialists
3. Specialists execute in parallel
4. Results merge into unified output
5. Expert Debate Engine evaluates (6 experts)
6. Chief Design Officer resolves conflicts
7. CEO makes final decision (if debate unresolved)
8. Results stored in Executive Memory
```

## 10-Phase Agency Workflow

```
Plan → SwarmCreation → ParallelExecution → Debate → Merge
→ Verification → Retry → Learning → Memory → Release
```

## Expert Debate Engine

6 experts evaluate every strategic decision:
1. Creative Director
2. SEO Specialist
3. Trust Architect
4. Conversion Director
5. Accessibility Lead
6. Performance Engineer

**Resolution**: Consensus → Chief Design Officer → CEO

## Agency First Principle Enforcement

```
Every strategic decision MUST pass 6 gates:
  Research → Expert Review → Debate → Evidence Verification → Executive Decision → Learning

If any gate fails → decision is REJECTED.
The platform optimizes for decision QUALITY, not generation SPEED.
```
