---

## Platform First Principle (최상위 설계 원칙)

```
Users do not visit websites to admire design.

Users visit websites to solve a problem.

Every page must answer the next question
inside the customer's mind.

Do not generate pages.

Generate customer decisions.
```

> **"Every section must earn its place. If a section does not answer a customer question, remove a fear, build trust, or move the customer to the next decision, it should not exist."**

이 원칙 하나만 지켜도 불필요한 배너, 의미 없는 카드, AI가 흔히 생성하는 장식성 섹션이 사라지고, 모든 페이지가 **고객의 실제 의사결정**을 중심으로 설계된다.

## 4대 Platform Rule

### Rule 01
고객은 페이지를 보러 오지 않는다.
**문제를 해결하기 위해 온다.**

### Rule 02
고객은 디자인을 평가하지 않는다.
**회사를 믿을 수 있는지 판단한다.**

### Rule 03
AI는 페이지를 생성하지 않는다.
**Decision Journey를 생성한다.**

### Rule 04
모든 Component는 시각적 목적이 아니라 **심리적 목적**을 가져야 한다.

| Component | 심리적 목적 |
|---|---|
| Hero | 첫인상 → 신뢰 → 다음 스크롤 |
| CTA | 예약 유도 |
| FAQ | 불안 제거 (Decision Accelerator) |
| Review | 사회적 증거 |
| About | 회사 신뢰 |
| Gallery | 실제 경험 증명 |

## 9-Stage Customer Journey (산업 공통)

```
Problem → Discovery → Comparison → Evaluation → Trust → Decision → Action → Experience → Loyalty → Advocacy
```

각 단계마다:
```
Goal
Question
Objection
Evidence
Decision Trigger
Next Action
```

## 11대 Framework (RC3.1 핵심)

### Framework 1. Customer Journey (9 stages)
산업 공통 여정 + 단계별 Goal/Question/Objection/Evidence/Trigger/NextAction

### Framework 2. Customer Psychology
각 Journey 단계의 심리 상태 — Goal/Question/Objection/Evidence/DecisionTrigger/NextAction

### Framework 3. Objection Library (산업별 반대 의견 라이브러리)
- Hostel: 안전한가, 여성 혼자, 늦은 체크인, 와이파이, 조식, 수건, 공항픽업
- Restaurant: 예약, 주차, 채식, 아이, 가격
- Travel: 환불, 비, 언어, 아이, 난이도
- SaaS: 가격, API, 보안, 도입, 마이그레이션
- Hospital: 의사, 경력, 비용, 보험, 장비
- Marketplace: 사기, 환불, 배송, 품질, 판매자

### Framework 4. Trust Evidence Placement
자동 결정: "무엇을 / 어디에 / 언제 / 보여줄 것인가"
수집이 아닌 **배치**가 목적

### Framework 5. Detail Strategy Library (12개 산업)
산업별 상세페이지 전략:
- Hospitality / Restaurant / Travel / Marketplace / Retail / Medical / Education / Real Estate / SaaS / NGO / Church / Government

각 Blueprint 정의:
- Section Order, Story Flow, Evidence Order, FAQ, CTA, Social Proof, Gallery, Objection, Comparison, Related Items, Cross Sell, Upsell

### Framework 6. FAQ Strategy (Decision Accelerator)
FAQ는 질문 목록이 아니다. **Decision Accelerator**이다.
각 FAQ는 다음 중 하나를 해결:
- Risk / Fear / Confusion / Comparison / Cost / Expectation / Support

### Framework 7. AI Concierge Framework
Chatbot 제거 → AI Concierge

역할:
```
현재 페이지
현재 Journey
현재 행동
현재 관심사
↓
다음 행동 추천
```

예) Hostel:
```
3명이시군요.
Dormitory와 Private 비교해 드릴까요?
공항픽업도 필요하신가요?
```

### Framework 8. Social Proof Architecture
Review만 사용 금지, 지원:
- Reviews / Awards / Media / Partners / Guest Stories / Instagram / YouTube / Google / TripAdvisor / Booking / Community / Press / Certification

### Framework 9. Story Architecture
모든 상세페이지 다음 Story 사용:
```
Emotion → Evidence → Trust → Decision → Action
```
정보 나열 금지

### Framework 10. Industry Detail Blueprint (12개 산업)
위 Framework 5의 Detail Strategy Library

### Framework 11. Customer Question Model (CQM) ⭐ 사장님 추가 권장

**"이 페이지에서 고객이 가장 궁금해할 질문 목록"을 먼저 생성하고, 그 질문을 가장 효과적으로 해결하는 순서로 콘텐츠와 컴포넌트를 배치한다.**

#### 예) Hostel Hero
```
안전한가?
위치는?
분위기는?
가격은?
후기는?
예약은?
```

#### 예) Booking 페이지
```
취소 가능한가?
환불은?
언제 결제?
체크인은?
추가 비용?
```

#### 예) About 페이지
```
이 회사는 누구인가?
왜 믿을 수 있는가?
왜 존재하는가?
```

**페이지는 Section으로 만드는 것이 아니라 Customer Question Sequence로 만들어진다.**

이것이 Amazon의 상품 상세, Booking.com의 숙소 페이지, Airbnb의 숙소 소개, 고급 호텔 사이트들이 실제로 하는 접근이다.
