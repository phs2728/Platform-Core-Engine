## 5대 엔진 RC3 영향

### Experience Engine
**이전**: 예쁜 Layout 만들기
**RC3**: 고객이 불안 → 신뢰 → 행동으로 이동하도록 설계

### Theme Engine
**이전**: 색상 생성
**RC3**: 브랜드 신뢰감 표현 (Luxury → 큰 여백 + 고급 사진 + 절제된 Motion + Typography → Trust)

### Component Engine
**이전**: Component를 만드는 것이 목표
**RC3**: 모든 Component는 Trust 목적을 가져야 함

| Component | Trust 목적 |
|---|---|
| Hero | 감성 → 신뢰 → 행동 |
| Review | 사회적 증거 |
| About | 회사 신뢰 |
| FAQ | 불안 제거 |
| CTA | 행동 유도 |

### CMS Engine
**이전**: 콘텐츠 저장
**RC3**: 콘텐츠가 Trust Architecture를 지원하는지 검증

- About에 회사 소개가 없으면 → Warning
- Review가 없으면 → Warning
- 실제 사진이 없으면 → Warning

### Studio Engine
**이전**: 웹사이트 생성
**RC3**: 신뢰 경험 생성

순서:
```
Research → Customer Psychology → Trust Architecture → Creative → Experience → Theme → Component → CMS → Publish
```

## 신규 산출물 (UI 대체)

대신 생성:
```
이 회사를 믿게 만드는 핵심 요소
  ↓
추천 배치
  ↓
왜 필요한지
  ↓
어느 페이지에 넣을지
```

**7대 신규 산출물 (자동 생성)**:
1. **Trust Architecture Report** — 산업별 신뢰 요소 매핑
2. **Customer Psychology Report** — 고객 의사결정 과정
3. **Evidence Placement Strategy** — Evidence를 어디에 배치할지
4. **Objection Map** — 고객 우려 사항과 해소 방법
5. **Confidence Journey** — 신뢰 형성 단계
6. **Decision Journey** — 행동 결정 단계
7. **Trust Checklist** — 최종 검증 (Evidence가 모두 배치되었는지)

## Platform Vision v2 — 최종 정의

Platform의 목적은:

> "예쁜 웹사이트를 만드는 것"

이 아니다.

Platform의 목적은:

> **"고객이 이 회사를 믿고, 문의하고, 예약하고, 구매하도록 심리적 신뢰 경험을 설계하는 것"**

이다.

---

## 차별점

현재 대부분의 AI 웹사이트 생성기:
- "레이아웃과 코드"를 생성

여러분의 Platform:
- **"고객의 의사결정 과정"**을 설계

Envoy 프로젝트에서는:
- "예쁜 부티크 호스텔 사이트"가 아니라
- **"방문자가 3초 안에 '이곳이라면 믿고 예약할 수 있겠다'고 느끼는 사이트"**

이것이 **앞으로 모든 산업(여행, 레스토랑, SaaS, 병원, 마켓플레이스)** 에서도 일관된 경쟁력이 된다.
